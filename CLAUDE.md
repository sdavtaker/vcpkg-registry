# vcpkg Registry

This repo is a private vcpkg git registry for sdavtaker/* libraries.
Layout follows the vcpkg git registry spec: `ports/<name>/` + `versions/`.

## Adding a New Port

1. Create `ports/<name>/vcpkg.json` — use `"version-semver"` for semver versions,
   never the untyped `"version"` field (vcpkg rejects it).
2. Create `ports/<name>/portfile.cmake` — see conventions below.
3. Commit the port files.
4. Run `vcpkg x-add-version --all --verbose` to generate/update
   `versions/<letter>-/<name>.json` and `versions/baseline.json`.
5. Commit the versions database separately.
6. Open a PR — the `release-validate` CI check must pass before merge.

## portfile.cmake Conventions

- Always set `set(VCPKG_BUILD_TYPE release)` at the top — these are header-only
  or release-only libraries; no debug variant is needed.
- Use `vcpkg_cmake_config_fixup()` after `vcpkg_cmake_install()` — NOT
  `vcpkg_cmake_config_integration()` (that function does not exist).
- Disable tests via the project's own option, not the generic `BUILD_TESTING`:
  each library exposes `<LIBNAME>_BUILD_TESTS=OFF`.
- SHA512 must be the real tarball hash — never use `0` as a placeholder.
  Compute it with: `curl -fsSL <tarball-url> | sha512sum`
- Do not add `file(REMOVE_RECURSE "${CURRENT_PACKAGES_DIR}/debug")` when
  `VCPKG_BUILD_TYPE release` is already set — the debug dir is never created.

## versions/ Database

- `versions/baseline.json` — maps each port name to its latest version.
- `versions/<first-letter>-/<name>.json` — full version history with git-tree SHAs.
- These files are generated by `vcpkg x-add-version`, not hand-edited.
- The git-tree SHAs here are distinct from the portfile SHA512 (tarball hash).

## Consumer Configuration

Consumers reference this registry via `vcpkg-configuration.json`:
```json
{
  "registries": [
    {
      "kind": "git",
      "repository": "https://github.com/sdavtaker/vcpkg-registry",
      "baseline": "<main-branch-commit-sha>",
      "packages": [ "<port-name>" ]
    }
  ]
}
```
The `baseline` value is the commit SHA of this registry's main branch at the
time the consumer's lockfile was last updated.
